01 Modal 元件(用 Portal + Esc 監聽)

最常被誤解的元件——必須用 createPortal 跳出父層,並監聽 Esc 鍵與背景點擊。

// Modal.tsx
import { useEffect } from 'react';
import { createPortal } from 'react-dom';

type Props = {
  open: boolean;
  onClose: () => void;
  onConfirm?: () => void;
  title: string;
  children: React.ReactNode;
};

export function Modal({ open, onClose, onConfirm, title, children }: Props) {
  // Esc 鍵全域監聽
  useEffect(() => {
    if (!open) return;
    const onKey = (e: KeyboardEvent) => {
      if (e.key === 'Escape') onClose();
    };
    window.addEventListener('keydown', onKey);
    return () => window.removeEventListener('keydown', onKey);
  }, [open, onClose]);

  if (!open) return null;

  return createPortal(
    <div className="backdrop" onClick={onClose}>
      <div className="modal" onClick={(e) => e.stopPropagation()}>
        <h3>{title}</h3>
        {children}
        <button onClick={onClose}>取消</button>
        <button onClick={() => { onConfirm?.(); onClose(); }}>確認</button>
      </div>
    </div>,
    document.body
  );
}
  • createPortal 跳出父層,避免被 overflow:hidden 截斷
  • e.stopPropagation() 防止點 modal 內部誤關
  • useEffect cleanup 清掉 listener,避免記憶體洩漏
02 Toast 系統(Provider + useToast hook)

用 Context 模式,任何元件呼叫 useToast() 就能跳通知。

// ToastProvider.tsx
import { createContext, useContext, useState, useCallback } from 'react';

type Toast = { id: string; msg: string; type: 'success'|'error' };
const Ctx = createContext<((msg: string, type?: string) => void) | null>(null);

export function ToastProvider({ children }) {
  const [toasts, setToasts] = useState<Toast[]>([]);

  const show = useCallback((msg, type = 'success') => {
    const id = crypto.randomUUID();
    setToasts((t) => [...t, { id, msg, type }]);
    setTimeout(() => setToasts((t) => t.filter((x) => x.id !== id)), 3000);
  }, []);

  return (
    <Ctx.Provider value={show}>
      {children}
      <div className="toast-container">
        {toasts.map((t) => (
          <div key={t.id} className={`toast ${t.type}`}>{t.msg}</div>
        ))}
      </div>
    </Ctx.Provider>
  );
}

export const useToast = () => {
  const ctx = useContext(Ctx);
  if (!ctx) throw new Error('useToast 必須包在 <ToastProvider> 內');
  return ctx;
};

// 使用
function SaveButton() {
  const toast = useToast();
  return <button onClick={() => toast('已儲存', 'success')}>儲存</button>;
}
  • Context 共享 toast 函式,避免 props drilling
  • useCallback 保證函式 reference 穩定,不會觸發子元件重渲染
  • throw 確保開發階段就抓到 Provider 沒包到的錯誤
03 自訂 Hooks(useDebounce / useLocalStorage)

把重複邏輯抽成 hook,跨元件重用。命名一律 use*。

// hooks.ts
import { useState, useEffect } from 'react';

// 1. useDebounce:延遲更新值,常用於搜尋輸入
export function useDebounce<T>(value: T, delay = 300): T {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(id);
  }, [value, delay]);
  return debounced;
}

// 2. useLocalStorage:雙向同步 localStorage
export function useLocalStorage<T>(key: string, initial: T) {
  const [value, setValue] = useState<T>(() => {
    try {
      const raw = localStorage.getItem(key);
      return raw ? JSON.parse(raw) : initial;
    } catch { return initial; }
  });
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  return [value, setValue] as const;
}

// 3. useMediaQuery:響應式偵測
export function useMediaQuery(query: string) {
  const [matches, setMatches] = useState(false);
  useEffect(() => {
    const m = window.matchMedia(query);
    setMatches(m.matches);
    const onChange = (e: MediaQueryListEvent) => setMatches(e.matches);
    m.addEventListener('change', onChange);
    return () => m.removeEventListener('change', onChange);
  }, [query]);
  return matches;
}

// 使用
const [query, setQuery] = useState('');
const debouncedQuery = useDebounce(query, 300);
const isMobile = useMediaQuery('(max-width: 768px)');
const [dark, setDark] = useLocalStorage('dark', false);
  • 泛型 <T> 保留型別資訊,使用端會自動推導
  • useEffect cleanup 一律寫,避免 stale closure 與洩漏
  • 用 as const 鎖定 tuple 型別,destructure 才不會變 union

Claude Code 是 Anthropic 的官方終端機編碼代理。推薦原生二進位,不需 Node.js。

# 推薦:原生二進位(macOS / Linux)
curl -fsSL https://claude.ai/install.sh | bash

# macOS Homebrew
brew install --cask claude-code

# 登入(瀏覽器)
claude auth login

# 用 Anthropic Console 計費
claude auth login --console

# CI 環境用長效 token
claude setup-token
  • 原生二進位最穩,不需 Node.js
  • 本機開發用訂閱登入即可
  • CI 用 setup-token 生成 token
02 啟動方式

互動模式適合來回對話,印出模式(-p)適合腳本與一次性查詢。

# 互動模式
claude

# 帶第一個問題進入
claude "解釋這個專案的架構"

# 一次性查詢(headless)
claude -p "這個 function 在做什麼?"

# 從 stdin 接收
git diff | claude -p "幫這份 diff 寫 commit message"

# 接續最近對話
claude -c

# 用名稱恢復 session
claude --resume "auth-refactor"
  • -p = print mode,適合腳本與 pipe
  • -c = 接續當前目錄最近對話
  • -r/--resume = 選擇歷史 session
03 模型 & 思考強度
# 指定模型
claude --model sonnet
claude --model opus
claude --model claude-sonnet-4-6

# 思考強度(low/medium/high/xhigh/max)
claude --effort high

# 主模型過載時自動 fallback
claude -p --fallback-model sonnet "query"

# 對話中切換模型
> /model opus
  • 複雜重構用 opus + --effort high
  • 日常編輯用 sonnet,速度快
  • fallback 只在 print mode 與背景任務生效
04 權限模式
# default - 每個敏感動作都詢問
claude --permission-mode default

# plan - 規劃模式(只想不做)
claude --permission-mode plan

# acceptEdits - 自動接受編輯
claude --permission-mode acceptEdits

# auto - 智慧自動判斷
claude --permission-mode auto

# bypass - 全部繞過(沙箱限定)
claude --dangerously-skip-permissions
  • 不熟悉專案先用 plan 模式
  • 互動中按 Shift+Tab 切換模式
  • bypass 只在 CI 或 sandbox 用
05 工具控制
# 只允許特定工具
claude --tools "Read,Edit,Bash"

# 預先核准某些指令
claude --allowedTools "Bash(git log *)" "Read"

# 禁用特定工具
claude --disallowedTools "Bash(rm *)"

# 新增可讀寫目錄
claude --add-dir ../shared-lib

# 完全唯讀(觀察模式)
claude --tools "Read,Grep,Glob"
  • code review 場景只給 Read 工具
  • 用 Bash(指令 *) 語法精細控制
  • 單一危險指令用黑名單
06 輸出格式(腳本化)
# JSON 輸出
claude -p --output-format json "query"

# 串流 JSON(即時)
claude -p --output-format stream-json --verbose

# 用 JSON Schema 約束輸出
claude -p --json-schema '{...}' "query"

# 最大成本上限(防爆預算)
claude -p --max-budget-usd 5.00

# 最大來回次數
claude -p --max-turns 3
  • CI/CD 用 json 取得結構化結果
  • --max-budget-usd 防無限執行
  • --json-schema 保證輸出可 parse
07 斜線指令(互動內)

在 claude 互動模式裡看到 > 提示符後輸入。

# 上下文管理
> /compact           // 壓縮歷史
> /context           // 查看 context 用量
> /clear             // 清空對話
> /cost              // 查看累計花費

# 工作目錄 & 設定
> /add-dir ../lib
> /init              // 產生 CLAUDE.md
> /memory            // 編輯記憶
> /review            // 對最近變更 review

# 模型 & 目標
> /model opus
> /goal "修復測試直到全綠"
  • /compact 在 context 警告時用
  • /goal 適合有明確完成條件的任務
  • 自訂指令放 .claude/commands/
08 背景任務

長時間任務丟背景,前景繼續工作。

# 啟動背景任務
claude --bg "重構整個 auth 模組"

# 列出全部背景任務
claude agents

# 看任務的即時 log
claude logs 7c5dcf5d

# 連接到任務終端
claude attach 7c5dcf5d

# 停止 / 重啟
claude stop 7c5dcf5d
claude respawn 7c5dcf5d
  • 大任務丟背景,前景繼續寫 code
  • respawn --all 換版本後一次重啟
  • 互動內 /agents 開監控面板
